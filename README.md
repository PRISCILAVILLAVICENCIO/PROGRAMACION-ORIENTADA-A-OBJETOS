Resumen Comparativo de Paradigmas
Enfoque Procedural: En este modelo, el código se organiza como una serie de pasos secuenciales y funciones que transforman datos. Es muy efectivo para tareas directas, ya que la lógica es fácil de seguir linealmente. Sin embargo, los datos están "sueltos", lo que puede generar errores si el programa se vuelve más grande.

Enfoque Orientado a Objetos (POO): Aquí el código se estructura en torno a "Clases", que funcionan como moldes. Hemos aplicado el encapsulamiento al hacer que la lista de temperaturas sea privada (__temperaturas), asegurando que solo los métodos de la clase puedan modificarla. También usamos herencia para crear un reporte más detallado sin alterar la clase original. Este paradigma es superior para proyectos escalables y para mantener el orden en el manejo de información sensible.
